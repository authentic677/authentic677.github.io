什么是系统调用

系统调用是操作系统提供对外提供服务的方式，应用程序通过系统调用请求操作系统完成特定功能

在C语言中，系统调用表现为一系列函数的调用，常见的系统调用函数如下

进程控制

  • **fork()**：创建一个新进程。

fork系统调用能让程序拥有多个执行流，提供了在程序中实现多任务的能力

fork后的父进程和子进程虽然拥有独立的内存空间(包括堆内存，使用写时复制(COW)技术实现)，但它们共享某些操作系统级别的资源，如底层文件描述符、网络套接字，例如如下情况：

fork后，socket本身并没有被克隆，只是它的引用被复制了，所以为避免互相干扰，各进程关闭多余的socket引用复制，只留自身需要的引用，避免出现并发安全问题。

  • **exec()**：执行一个程序（有多个变种，如execl()、execp()等）。

子程序

父程序

execl(): 这是exec系列调用的一种，参数包括程序路径、参数列表，参数列表以NULL结尾。

"./child": 指定要执行的可执行文件（child）。

"child": 传递给子程序的第一个参数，通常是程序名（argv[0]）。

Shell（如bash、zsh等）使用exec系统调用来执行用户输入的命令。例如，当你在命令行中输入一个命令时，Shell解析命令后，会使用fork()创建一个子进程，然后在子进程中调用exec来执行该命令对应的程序。exec会将当前进程的内容替换为新程序，而不会创建新的进程，这节省了系统资源。

  • **exit()**：终止进程并返回状态码。
  • **wait()**：等待子进程结束。

应用场景：一个父进程创建多个子进程，每个子进程负责处理一个独立的任务。父进程使用wait()逐一等待所有子进程完成后，再收集结果或继续执行后续任务

wait()系统调用会使父进程阻塞，直到它的任意一个子进程终止。wait()返回的值是结束的子进程的进程ID

  • **getpid()**：获取当前进程ID。
  • **getppid()**：获取父进程ID。
  • **kill()**：向进程发送信号。

文件操作

  • **open()**：打开文件。
  • **read()**：从文件中读取数据。

函数原型

参数

**fd**：要读取的文件描述符。这个文件描述符通常是由 open 函数返回的，或者是标准输入、输出和错误流（如 stdin、stdout、stderr）的文件描述符。

**buf**：指向一个内存缓冲区的指针，用于存储读取的数据。

**count**：要读取的最大字节数。

返回值

成功：返回实际读取的字节数。如果返回值小于 count，则表示读取的字节数少于请求的字节数，可能是因为到达了文件末尾或缓冲区已满。

失败：返回 -1，并设置 errno 以指示错误类型。常见的错误包括 EBADF（文件描述符无效）、EFAULT（缓冲区指针无效）和 EINTR（操作被中断）。

read函数会阻塞直到有可获得的数据

  • **write()**：向文件中写入数据。
  • **close()**：关闭文件描述符。
  • **lseek()**：移动文件指针。
  • **unlink()**：删除文件。
  • **stat()**：获取文件状态信息。
  • **fstat()**：获取文件描述符的状态信息。
  • **lstat()**：获取文件状态信息（处理符号链接）。

3. 内存管理

  • **mmap()**：映射文件到内存。
  • **munmap()**：解除映射。
  • **brk()**：调整数据段大小（用于堆）。
  • **sbrk()**：增加或减少堆的大小。

4. 设备操作

  • **ioctl()**：控制设备行为。

ioctl是input output control的缩写，是一个非常强大的系统调用函数，能做很多需要root权限的操作

例子：获取网络接口的ip地址

网络接口相关的 ioctl 命令

  1. **SIOCGIFNAME**：获取接口名称。
  2. **SIOCGIFBRDADDR**：获取广播地址。
  3. **SIOCGIFNETMASK**：获取网络掩码。
  4. **SIOCGIFHWADDR**：获取硬件地址（MAC 地址）。
  5. **SIOCGIFFLAGS**：获取接口标志（如是否启用）。
  6. **SIOCSIFFLAGS**：设置接口标志（如启用或禁用接口）。
  7. **SIOCGIFMTU**：获取接口的最大传输单元（MTU）。
  8. **SIOCSIFMTU**：设置接口的最大传输单元（MTU）。

重磅级ioctl命令**TUNSETIFF**(TUN SET Interface Flags)

5. 网络通信

  • **socket()**：创建一个套接字。
  • **bind()**：绑定套接字到一个地址。
  • **listen()**：监听套接字上的连接请求。
  • **accept()**：接受连接请求。
  • **connect()**：连接到一个远程主机。
  • **send()**：发送数据。
  • **recv()**：接收数据。
  • **sendto()**：发送数据到指定地址。
  • **recvfrom()**：从指定地址接收数据。
  • **shutdown()**：关闭套接字的某些功能。
  • **getsockname()**：获取套接字的本地地址。
  • **getpeername()**：获取套接字的对端地址。

  • **pipe()**：创建管道。

管道适用于父子进程间的通信

  • **mkfifo()**：创建FIFO（命名管道）。

7. 文件系统操作

  • **mount()**：挂载文件系统。
  • **umount()**：卸载文件系统。

8. 权限管理

  • **chmod()**：更改文件权限。
  • **chown()**：更改文件所有者。
